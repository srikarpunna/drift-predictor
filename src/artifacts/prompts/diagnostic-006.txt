This run set isolates nondeterministic behavior in the new model. The original 5 prompt groups show: P-601 (2/3 inconsistency on candidate_eval), P-602 (1/3 inconsistency with severe "pass"→"fail" flip), P-603 (2/3 inconsistency on risk_assessment), P-604 (stable, 3/3 consistent passing), P-605 (2/3 inconsistency on salary_benchmark). The expanded battery adds new prompts in the same families plus two new families (performance_tier, compensation_band), all of which exhibit the same boundary-oscillation pattern.

The consistency failures cluster around prompts with borderline inputs: a candidate with moderate scores (P-601, P-602, P-606), an employment gap of ambiguous severity (P-603, P-607), a salary at a boundary percentile (P-605, P-608), a performance score near a tier threshold (P-609), and a compensation total $2K above a band boundary (P-610). P-604 was the only stable prompt — it had clear-cut low-risk indicators with no decision boundary ambiguity. This confirms the borderline-input hypothesis: new model consistency failures are triggered specifically by near-threshold inputs.

The cross-family pattern is consistent: in every case where a new triplet shows inconsistency (P-606, P-607, P-608, P-609, P-610), the input was designed to sit within 5% of a classification boundary. The old model resolved all these cases deterministically, suggesting its internal thresholds were better calibrated to produce stable outputs at the boundary values present in these prompts. The new model's temperature-induced sampling variation activates near-boundary inputs and is not present on clear-cut inputs (P-604 equivalent prompts).

Excluding no pre-existing failures (all 28 runs have old_passed=YES), 15 regressions occurred on 28 previously passing runs — a 53.6% regression rate distributed across all five task families. No family shows a zero regression rate, confirming this is a model-level sensitivity to borderline inputs rather than a family-specific prompt issue.
