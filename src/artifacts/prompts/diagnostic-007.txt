    <pattern_analysis>
This run set reveals two distinct consistency failure subtypes. The first is stochastic boundary oscillation: the new model's classification output for boundary-case inputs varies across runs due to temperature-induced sampling variation near internal decision thresholds. P-701 ($42K ARR near $40K boundary), P-703 (moderate-severity clause), P-705 (71% match near 70% threshold), P-707 ($43.5K ARR near $40K boundary), P-708 (moderate liability clause), P-709 (72% match near 70% threshold), P-710 (grade boundary case), P-711 (4.2/5.0 near 4.0 boundary) all exhibit 1/2 or 2/3 run inconsistency. The old model resolved all these cases deterministically.

The second subtype, exemplified by P-706, is systematic miscalibration: all 3 runs produce the same wrong answer. A 68% match score returning "strong" is not boundary ambiguity — it is a threshold shift in the new model's internal calibration. Unlike stochastic oscillation (which would self-correct with lower temperature), systematic miscalibration requires the model's scoring calibration to be corrected in the prompt or via fine-tuning.

The two new task families — pay_grade_assessment (R-025 through R-027) and engagement_level (R-028 through R-029) — both exhibit the boundary_oscillation pattern. P-710 shows 2/3 inconsistency on a grade boundary case; P-711 shows 1/2 inconsistency on a near-threshold engagement score. This cross-family consistency confirms the failure mechanism is model-level sensitivity to near-boundary inputs, not prompt-specific.

No pre-existing failures exist in this dataset (all 29 runs have old_passed=YES). 15 regressions occurred on 29 previously passing runs — a 51.7% regression rate. Talent_matching leads with 6 regressions (75% rate across 8 rows), followed by contract_review with 3 regressions (37.5%), tiered_pricing with 3 regressions (37.5%), pay_grade_assessment with 2 regressions (66.7%), and engagement_level with 1 regression (50.0%).
