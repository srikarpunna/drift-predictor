    <pattern_analysis>
This run set exhibits two distinct schema_adherence failure subtypes. The first is structural flattening: the new model collapses hierarchical objects into simpler representations (R-001, R-003, R-006, R-008, R-016, R-017, R-019, R-022, R-026). In each case the semantic content is preserved but the structure is simplified — arrays of objects become strings, objects become scalars, lists become prose. This is consistent with the model prioritizing output brevity over strict schema compliance.

The second subtype is missing required fields and partial object returns: the new model omits keys entirely (R-010, R-024) or returns objects with required subfields absent (R-012, R-021, R-028, R-030). R-012 drops `accrual_rate` from within `pto_policy`; R-021 adds an extra nesting layer while dropping `annual_allowance`; R-030 returns only the `health_fsa` portion of `fsa_options`. These partial-object failures are more subtle than complete key absence because they pass a superficial presence check while failing at subfield validation.

Two new task families added in this expanded run set — offer_letter_parse (R-022 through R-025) and compensation_breakdown (R-026 through R-029) — exhibit the same failure distribution as the original three families. Offer_letter_parse shows structural flattening (R-022) and field omission (R-024). Compensation_breakdown shows object-to-scalar collapse (R-026) and null-for-object regression (R-028). This cross-family consistency confirms that the failure mechanism is model-level behavioral regression rather than any prompt-specific issue.

Excluding R-011 (pre-existing failure, old_passed=NO), the regression count is 17 out of 29 previously-passing runs — a 58.6% regression rate.
